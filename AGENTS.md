# Agent Guide — `mussab-project`

> This file is written for AI coding agents. It assumes you know Unity/C# but know nothing about this specific project. All facts below were derived from the actual repository contents (manifests, project settings, source code, and documentation).

## Project overview

- **Product name:** `mussab-project`
- **Company:** `DefaultCompany`
- **Bundle version:** `0.1.0`
- **Engine:** Unity `6000.5.1f1` (Unity 6)
- **Template origin:** `com.unity.template.urp-blank@17.0.14`
- **Genre / goal:** First-person psychological horror walking simulator set in a small Libyan residential neighborhood. The current focus is the *environment greybox*; there is no combat, inventory, dialogue, or AI at this stage.
- **Design documents:**
  - `Docs/WorldDesign.md` — overall neighborhood layout, tone, and environmental rules.
  - `Docs/AbandonedHouseDesign.md` — detailed courtyard-house level design that supersedes parts of `WorldDesign.md` where they conflict.

The project is intentionally small and art-light. The world is built from primitive shapes and a curated greybox material palette. All horror beats are expected to come from lighting, sound, and spatial arrangement rather than code-driven events.

- **Runtime code lives in `Assets/Scripts/`.** Currently the only runtime script is `PlayerController.cs`.
- **Editor-only code lives in `Assets/Editor/`.** `NeighborhoodWorldBuilder.cs` is an editor tool that builds the entire `Neighborhood.unity` scene from primitives and configures lights, fog, and the player rig.
- **Generated project files:** `Assembly-CSharp.csproj`, `Assembly-CSharp-Editor.csproj`, and `mussab-project.slnx` are generated by Unity. Do not hand-edit them; changes are overwritten when Unity regenerates them.
- **Asset serialization:** `EditorSettings.asset` sets `m_SerializationMode: 2` (Force Text), so scenes, prefabs, and materials are stored as YAML text, not binary.

## Runtime architecture

### Player rig

`NeighborhoodWorldBuilder.CreatePlayerRig()` creates the playable first-person rig at runtime-in-editor:

- A `Player` GameObject with:
  - `CharacterController` (height 1.8, radius 0.3, step offset 0.35).
  - `PlayerController` script.
  - Tag `Player`.
- A child `Main Camera` with:
  - `Camera` (near 0.05, far 400, solid-color clear).
  - `AudioListener`.
  - Tag `MainCamera`.
- A child `Flashlight` SpotLight (warm color `#F2C98C`, toggled with `F`).

### PlayerController

`Assets/Scripts/PlayerController.cs`:

- Uses the **New Input System via direct device polling** (`Keyboard.current`, `Mouse.current`), not the `InputSystem_Actions` asset.
- Controls:
  - `WASD` / arrow keys — move
  - `Shift` — sprint
  - `Space` — jump
  - Mouse — look
  - `F` — toggle flashlight
  - `Esc` — release cursor
  - Left-click — recapture cursor
- Uses `CharacterController.Move()` with a custom gravity value (`-18`).
- Cursor locking is handled manually in `Update()`.

### World / lighting

- The `Neighborhood.unity` scene is generated by the builder.
- It includes a main road, alley, family house, shop, school, parked cars, street lamps, and the abandoned courtyard house (the centerpiece level).
- Global lighting is configured in code by the builder:
  - One directional "moon" light (cool `#A8BBD4`, soft shadows, intensity 0.35).
  - Flat ambient color `#1E222A`.
  - Exponential fog (`#151A20`, density 0.012).
- Quality settings define two levels: `Mobile` and `PC`. `PC` is the default for `Standalone`.

## Code style guidelines

Observed conventions in the existing code:

- **Language:** All comments and documentation are in English.
- **Naming:**
  - `PascalCase` for classes, methods, and constants (e.g., `NeighborhoodWorldBuilder`, `BuildAbandonedHouse`, `ScenePath`).
  - `camelCase` for fields and locals (e.g., `walkSpeed`, `flashlight`, `pitch`).
  - Unity-style component references use lowercase fields initialized in `Awake()`/`Start()`.
- **Access modifiers:** Explicit `public` for serialized/inspectable fields; `private`/`static` helpers omitted where default is acceptable.
- **Organization:** Large editor tool is split into region-like sections (`// ground`, `// abandoned house`, `// materials`) rather than using `#region`.
- **Magic numbers:** The builder contains many literal coordinates. When extending it, prefer named constants (the abandoned-house section already uses `LOW`, `HIGH`, `LOWTOP`, `HIGHTOP`).
- **No unsafe code:** `ProjectSettings/ProjectSettings.asset` has `allowUnsafeCode: 0`.
- **Do not edit generated `.csproj`/`.slnx` files directly.** If IDE integration is wrong, regenerate by opening the project in Unity.

## Important editor-tool behavior

`NeighborhoodWorldBuilder` runs inside the Unity Editor only:

- `[InitializeOnLoadMethod]` triggers once per editor session if `Assets/Scenes/Neighborhood.unity` does not exist; it then builds the scene automatically.
- Menu items under **Tools > Neighborhood**:
  - **Build World (overwrites Neighborhood scene)** — rebuilds `Neighborhood.unity` from scratch. This is destructive to that scene; it creates a new empty scene and regenerates everything.
  - **Add Player To Current Scene** — removes old `Main Camera`/`Player`/`PlayerStart` objects and injects the playable rig into whichever scene is currently open.
- The builder creates greybox materials under `Assets/Art/Materials/Greybox/` if they do not already exist.

## Security considerations

- The project contains **no networking code, no third-party SDKs, and no credential files**.
- The only editor script that writes to disk (`NeighborhoodWorldBuilder`) creates scene/material assets under the project folder and logs its actions.
- `.gitignore` follows the standard Unity template and excludes `Library/`, `Temp/`, `Logs/`, `Build/`, `UserSettings/`, and generated binaries. Do not commit those folders.
- No secrets or API keys were found in source, YAML, or JSON files.

## Quick reference for agents

- Open in Unity `6000.5.1f1`.
- If `Neighborhood.unity` is missing, reopen the editor or run **Tools > Neighborhood > Build World**.
- To test walking the environment, open `Neighborhood.unity` and press Play.
- To ship a build, add `Neighborhood.unity` to **File > Build Settings** and choose the desired target platform.
- Keep changes minimal and consistent with the existing greybox/prototype style. New gameplay systems should be added behind clear, well-named components in `Assets/Scripts/`.
